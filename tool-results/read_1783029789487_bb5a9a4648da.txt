    34→## 1. Vue d'ensemble
    35→
    36→### Philosophie du pipeline
    37→
    38→Le menu **Stratégie** de CreaPulse V2 est construit autour d'un **pipeline de données en entonnoir** où chaque module produit des données qui alimentent les modules suivants :
    39→
    40→```
    41→┌─────────────────────────────────────────────────────────────────┐
    42→│                     PARCOURS (Amont)                             │
    43→│  Mon Projet ──→ Vision ──→ Profil Créateur                      │
    44→│  (Projet, Secteur, Cible)  (Objectifs, Valeurs)                 │
    45→└──────────────────────┬──────────────────────────────────────────┘
    46→                       │  "Générer depuis le Parcours"
    47→                       ▼
    48→┌─────────────────────────────────────────────────────────────────┐
    49→│                  STRATÉGIE — Simulateurs                         │
    50→│  Marché ──→ Juridique ──→ Financier ──→ CreaSim                 │
    51→│  (Marché, SWOT)   (Statut)   (3 ans)      (Marges mensuelles)   │
    52→└──────────────────────┬──────────────────────────────────────────┘
    53→                       │  "Synchroniser les simulateurs"
    54→                       ▼
    55→┌─────────────────────────────────────────────────────────────────┐
    56→│              BUSINESS PLAN (Hub central — 24 sections)            │
    57→│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐           │
    58→│  │Présenta. │ │  Marché  │ │ Finances │ │Opérations│           │
    59→│  │ 5 sect.  │ │  6 sect. │ │  6 sect. │ │  7 sect. │           │
    60→│  └──────────┘ └──────────┘ └──────────┘ └──────────┘           │
    61→└──────┬───────────────────┬──────────────────────┬───────────────┘
    62→       │                   │                      │
    63→       ▼                   ▼                      ▼
    64→┌──────────────┐   ┌──────────────┐       ┌──────────────┐
    65→│  BMC Canvas   │   │  Pitch Deck  │       │  Export PDF  │
    66→│  9 blocs IA  │   │  8 slides IA │       │  PPTX / PDF  │
    67→└──────────────┘   └──────────────┘       └──────────────┘
    68→```
    69→
    70→### Flux utilisateur typique
    71→
    72→1. **Parcours** : L'utilisateur remplit son projet (Mon Projet) et sa vision → alimentation automatique du BP
    73→2. **Stratégie — Simulateurs** : L'utilisateur utilise les 4 simulateurs (Marché, Juridique, Financier, CreaSim) → synchronisation dans le BP
    74→3. **Business Plan** : L'utilisateur finalise les 24 sections, avec suggestions IA par section
    75→4. **BMC + Pitch Deck** : Génération automatique depuis le BP finalisé, avec raffinement IA
    76→5. **Exports** : PDF du BP, HTML/PDF du BMC, PPTX du Pitch Deck

    80→## 2. Organisation logique du menu Stratégie
    81→
    82→### Navigation dans le Bureau Virtuel
    83→
    84→Le Bureau Virtuel est organisé en **5 sections principales**, chacune contenant des modules :
    85→
    86→| Section | Icon | Modules | Rôle |
    87→|---------|------|---------|-----|
    88→| **Parcours** | `Compass` | profil-createur, mon-projet, vision, pepites, riasec, kiviat, bilan-ia, creascope | Connaître le porteur et son projet |
    89→| **Stratégie** | `Target` | marche, juridique, financier, creasim, bmc, business-plan, pitch-deck | Construire le business |
    90→| **Écosystème** | `Network` | annuaire, forum, messages, mentorat | S'entourer |
    91→| **Pilotage** | `Gauge` | tremplin, passeport, certifications, téléchargements, vie privée | Piloter et certifier |
    92→
    93→### Ordre recommandé dans la section Stratégie
    94→
    95→L'ordre des modules dans le sidebar est **intentionnellement ordonnancé** pour refléter le pipeline :
    96→
    97→```
    98→1. 📊 Marché         → Comprendre le marché (SWOT, concurrents, segmentation)
    99→2. ⚖️  Juridique      → Choisir son statut légal (SAS, SARL, EURL, etc.)
   100→3. 💰 Financier      → Projeter sur 3 ans (CA, charges, rentabilité)
   101→4. 📈 CreaSim (IA)   → Simuler au mois le mois (marges, seuil de rentabilité)
   102→5. 🧩 BMC            → Synthétiser en 9 blocs le modèle économique
   103→6. 📄 Business Plan  → Rédiger le document complet (24 sections)
   104→7. 🎤 Pitch Deck     → Préparer la présentation investisseurs (8 slides)
   105→```
   106→
   107→**Logique** : Les modules 1-4 sont des **simulateurs d'entrée** qui produisent des données. Les modules 5-7 sont des **documents de sortie** qui consomment ces données.
   108→
   109→### Vue d'ensemble du menu (SectionOverview)
   110→
   111→Quand l'utilisateur clique sur "Stratégie" sans sélectionner de module, une grille de **7 cartes** s'affiche montrant chaque module avec :
   112→- Son nom et icône
   113→- Son état de complétion (via l'API `/api/progress`)
   114→- Un bouton d'accès direct

   118→## 3. Les briques d'alimentation (Parcours)
   119→
   120→Avant d'entrer dans la Stratégie, le **Parcours** du créateur alimente le pipeline avec des données fondamentales.
   121→
   122→### 3.1 Mon Projet (`mon-projet.tsx`)
   123→
   124→**5 étapes du formulaire :**
   125→
   126→| Étape | Champs | Destination dans le BP |
   127→|-------|--------|------------------------|
   128→| 1. Identité | projectTitle, projectSector, projectDescription, projectStage | `resume`, `historique` |
   129→| 2. Marché | primaryTarget, secondaryTarget, problemSolved, competitiveAdvantage, marketSize | `segmentation`, `etude-marche` |
   130→| 3. Économie | revenueSources[], revenueYear1/2/3, initialInvestment, financingNeed | `financement`, `compte-resultat` |
   131→| 4. Équipe | teamType, associateCount, motivation, keyCompetencies[] | `equipe`, `associes` |
   132→| 5. Résumé | Score de maturité /100, récapitulatif complet | — |
   133→
   134→**API** : `GET/PUT /api/projet` → stocké dans `CreatorJourney`
   135→
   136→### 3.2 Vision (`vision.tsx`)
   137→
   138→| Section | Champs | Destination dans le BP |
   139→|---------|--------|------------------------|
   140→| Énoncé de vision | visionStatement (vision à 5 ans) | `vision` |
   141→| Motivation & Impact | motivation, desiredImpact | `valeurs` |
   142→| Objectifs stratégiques | objectives[] (title, description, priority, deadline) | `vision` |
   143→| Valeurs fondamentales | coreValues[] (max 5 parmi 15 prédéfinies) | `valeurs` |
   144→| Jalons | milestones (6 mois, 1 an, 3 ans, 5 ans) | `calendrier` |
   145→
   146→**API** : `GET/PUT /api/vision` → stocké dans `CreatorJourney.visionAnswers`
   147→
   148→### 3.3 Action "Générer depuis le Parcours"
   149→
   150→Dans l'éditeur Business Plan, le bouton **"Générer depuis le Parcours"** appelle `POST /api/business-plan` avec `action: 'generate-from-parcours'`. Cela :
   151→
   152→1. Récupère en parallèle : `User`, `Beneficiary`, `CreatorJourney`, `RiasecResult[]`, `KiviatResult[]`, `ModuleResult[]`
   153→2. Construit un prompt LLM avec tout ce contexte
   154→3. Génère 8 sections BP (resume, equipe, historique, vision, valeurs, + 3 sections projet)
   155→4. **Ne surcharge pas** les sections déjà remplies — fusion seulement

   163→### 4.1 Marché (Analyse de Marché)
   164→
   165→**Fichier** : `src/components/bureau/modules/marche.tsx` (~895 lignes)  
   166→**API** : `GET/PUT/POST /api/marche`  
   167→**LocalStorage** : `creapulse-marche`
   168→
   169→#### Interface utilisateur
   170→
   171→- **4 cartes KPI** : CA potentiel, Marge concurrentielle, Coût acquisition client, Score attractivité
   172→- **Camembert (Recharts)** : Répartition des parts de marché
   173→- **5 sliders de simulateur** : Taille du marché, Part de marché, Concurrents, Budget marketing, Potentiel de croissance
   174→- **Jauge SVG animée** : Indicateur de croissance
   175→- **SWOT 4 quadrants** : Forces / Faiblesses / Opportunités / Menaces (4 textareas)
   176→- **Synthèse IA** : Carte affichant l'analyse générée par l'IA
   177→
   178→#### Types de données
   179→
   180→```typescript
   181→interface SimulateurData {
   182→  tailleMarche: number      // défaut: 1 000 000 €
   183→  partDeMarche: number      // défaut: 5 %
   184→  concurrents: number       // défaut: 10
   185→  budgetMarketing: number    // défaut: 5 000 €/mois
   186→  potentielCroissance: number // défaut: 50 %
   187→}
   188→
   189→interface SwotData {
   190→  strengths: string
   191→  weaknesses: string
   192→  opportunities: string
   193→  threats: string
   194→}
   195→```
   196→
   197→#### Actions IA
   198→
   199→- `POST /api/marche` avec `action: 'ai-analyze-market'` → Analyse complète du marché par l'IA
   200→- `POST /api/marche` avec `action: 'ai-autofill'` → Remplissage automatique de tous les champs
   201→
   202→#### Synchronisation vers le BP
   203→
   204→À chaque sauvegarde, le module appelle `PUT /api/business-plan` avec `action: 'sync-simulators'`, ce qui injecte les données dans les sections BP :
   205→- `etude-marche` ← secteur, taille du marché, potentiel
   206→- `segmentation` ← cible
   207→- `concurrence` ← nombre de concurrents, avantage compétitif
   208→- `strategie-marketing` ← budget marketing, part de marché
   209→- `plan-commercial` ← données commerciales
   210→- `swot` ← les 4 quadrants SWOT

   214→### 4.2 Juridique (Analyse Juridique)
   215→
   216→**Fichier** : `src/components/bureau/modules/juridique.tsx` (~905 lignes)  
   217→**API** : `GET/PUT/POST /api/juridique`  
   218→**LocalStorage** : `creapulse-juridique`
   219→
   220→#### Interface utilisateur
   221→
   222→- **5 cartes de statut** : SAS, SARL, EURL, Auto-entrepreneur, SASU (sélection visuelle)
   223→- **4 sliders** : CA prévisionnel, Nombre d'associés, Capital social, Réduction ACRE
   224→- **4 cartes résultat** : Régime fiscal, Charges sociales, Plafond TVA, Éligibilité ACRE
   225→- **Graphique comparatif (Recharts)** : Barres de comparaison des charges sociales
   226→- **Carte détaillée** : Avantages/Désavantages du statut sélectionné
   227→
   228→#### Types de données
   229→
   230→```typescript
   231→type StatutJuridique = 'SAS' | 'SARL' | 'EURL' | 'Auto-entrepreneur' | 'SASU'
   232→
   233→interface SimulateurJuridique {
   234→  statut: StatutJuridique      // défaut: SARL
   235→  caPrev: number              // défaut: 50 000 €
   236→  associes: number            // défaut: 2
   237→  capitalSocial: number        // défaut: 1 000 €
   238→  reductionCharges: number    // défaut: 50 %
   239→}
   240→```
   241→
   242→#### Moteur de recommandation
   243→
   244→Le module inclut un **moteur de scoring basé sur des règles** (pas d'IA) qui évalue 5 statuts selon 7 critères :
   245→
   246→| Critère | Question |
   247→|---------|----------|
   248→| Type d'activité | Commerciale, artisanale, libérale, services |
   249→| Nombre d'associés | Solo, 2-10, 10+ |
   250→| Capital initial | <1 000 €, 1 000-10 000 €, >10 000 € |
   251→| CA prévisionnel | <50 000 €, 50 000-150 000 €, >150 000 € |
   252→| Préférence de responsabilité | Limitée, illimitée |
   253→| Régime social | TNS, Assimilé-salarié |
   254→| Plans de croissance | Stable, modéré, ambitieux |
   255→
   256→#### Actions IA
   257→
   258→- `POST /api/juridique` avec `action: 'ai-suggest'` → Suggestion pour une question spécifique
   259→- `POST /api/juridique` avec `action: 'ai-autofill'` → Remplissage automatique des 7 questions
   260→
   261→#### Synchronisation vers le BP
   262→
   263→Le module synchronise les données juridiques dans la section BP `statut-juridique` (statut recommandé, régime fiscal, charges sociales, TVA, ACRE).

   267→### 4.3 Financier (Prévisions Financières)
   268→
   269→**Fichier** : `src/components/bureau/modules/financier.tsx` (~747 lignes)  
   270→**API** : `GET/PUT/POST /api/financier`  
   271→**LocalStorage** : `creapulse-financier-sim`
   272→
   273→#### Interface utilisateur
   274→
   275→- **5 cartes KPI** : CA Année 1/2, Charges A1, Résultat A1, Investissement, Cumulé 3 ans
   276→- **4 sliders** : CA A1, Croissance annuelle, Charges A1, Investissement
   277→- **Calculs en temps réel** : Marge nette A1/A3, Seuil de rentabilité
   278→- **3 jauges de rentabilité (Recharts RadialBarChart)**
   279→- **Graphique barres 3 ans** : CA vs Charges par année
   280→- **Tableau de synthèse 3 ans**
   281→
   282→#### Types de données
   283→
   284→```typescript
   285→// 4 entrées (sliders)
   286→year1Revenue: number       // défaut: 100 000 €
   287→growthRate: number         // défaut: 15 %
   288→year1Expenses: number      // défaut: 70 000 €
   289→initialInvestment: number  // défaut: 30 000 €
   290→
   291→// Calculs automatiques
   292→year2Revenue = year1Revenue × (1 + growthRate/100)
   293→year3Revenue = year2Revenue × (1 + growthRate/100)
   294→expenseGrowth = growthRate / 200
   295→year2Expenses = year1Expenses × (1 + expenseGrowth)
   296→year3Expenses = year2Expenses × (1 + expenseGrowth)
   297→margeNetteA1 = (year1Revenue - year1Expenses) / year1Revenue × 100
   298→seuilRentabiliteMois = initialInvestment / margeNetteMensuelle
   299→```
   300→
   301→#### Synchronisation vers le BP
   302→
   303→Les projections financières synchronisent **5 sections BP** :
   304→- `financement` ← Plan de financement (sources + montants)
   305→- `compte-resultat` ← CA et charges année 1/2/3
   306→- `tresorerie` ← Plan de trésorerie mensuel
   307→- `seuil-rentabilite` ← Seuil de rentabilité calculé
   308→- `bilan` ← Bilan prévisionnel

   312→### 4.4 CreaSim (Simulateur Financier Mensuel)
   313→
   314→**Fichier** : `src/components/bureau/modules/creasim.tsx` (~1 050 lignes)  
   315→**API** : `GET/POST /api/creasim`  
   316→**Badge** : `IA` dans le sidebar
   317→
   318→#### Interface utilisateur
   319→
   320→- **5 cartes KPI trafic** : Marge brute, Marge nette, Seuil de rentabilité, CA mensuel, Rentabilité A1
   321→- **7 sliders** : CA mensuel, Charges fixes (ajoutables/supprimables), Taux charges variables, Prix de vente unitaire, Coût unitaire, Investissement initial, Taux de marge cible
   322→- **4 jauges SVG circulaires**
   323→- **Graphique aires 12 mois (Recharts)** : CA vs Charges mensuelles
   324→- **3 cartes de rentabilité annuelle** : CA/Charges/Marge par année
   325→
   326→#### Types de données
   327→
   328→```typescript
   329→interface SimulationInputs {
   330→  monthlyRevenue: number         // défaut: 5 000 €
   331→  fixedCharges: FixedCharge[]     // défaut: [{Loyer: 800}, {Assurances: 150}, {Abonnements: 100}]
   332→  variableChargesRate: number    // défaut: 30 %
   333→  averageSellingPrice: number    // défaut: 100 €
   334→  unitCost: number               // défaut: 40 €
   335→  initialInvestment: number      // défaut: 15 000 €
   336→  targetMarginRate: number       // défaut: 40 %
   337→}
   338→
   339→interface SimulationResults {
   340→  fixedChargesTotal: number
   341→  variableChargesAmount: number
   342→  totalCharges: number
   343→  grossMarginAmount: number
   344→  grossMarginRate: number
   345→  netMarginAmount: number
   346→  netMarginRate: number
   347→  monthlyBreakeven: number       // CA minimum pour couvrir les charges
   348→  breakevenMonths: number        // Mois pour atteindre le seuil
   349→  profitability1Y/2Y/3Y: number  // Rentabilité cumulée
   350→  year1/2/3Revenue: number
   351→  year1/2/3Expenses: number
   352→}
   353→```
   354→
   355→#### Synchronisation vers le BP
   356→
   357→Le CreaSim synchronise dans la section BP `seuil-rentabilite` avec les données de marge et seuil calculés au mois le mois. Il alimente aussi les projections 3 ans dans les sections financières du BP.

   361→### 4.5 Business Model Canvas (BMC)
   362→
   363→**Fichier** : `src/components/bureau/modules/bmc.tsx` (~495 lignes)  
   364→**API** : `GET/PUT/POST /api/bmc`  
   365→**Badge** : `Nouveau` dans le sidebar
   366→
   367→#### Interface utilisateur
   368→
   369→- **Grille 9 blocs** en layout standard du BMC :
   370→  ```
   371→  ┌──────────────┬──────────────┬──────────────┐
   372→  │ Partenaires  │  Activités   │  Ressources  │
   373→  │    Clés      │    Clés      │    Clés      │
   374→  ├──────────────┴──────────────┴──────────────┤
   375→  │          Proposition de Valeur              │
   376→  ├──────────────┬──────────────┬──────────────┤
   377→  │  Relations   │   Canaux     │  Segments    │
   378→  │   Clients    │              │   Clients    │
   379→  ├──────────────┴──────────────┴──────────────┤
   380→  │ Structure des Coûts │ Sources de Revenus   │
   381→  └─────────────────────┴──────────────────────┘
   382→  ```
   383→- **Indicateur par bloc** : ✓ (rempli) ou ○ (vide) + compteur de caractères
   384→- **Badge de statut** : `DRAFT` → `GENERATED` → `REFINED`
   385→- **Auto-save** au blur avec debounce de 1.5 secondes
   386→
   387→#### Les 9 blocs
   388→
   389→| ID Frontend | Clé Backend | Titre |
   390→|-------------|-------------|-------|
   391→| `partenaires-cles` | `partenairesCles` | Partenaires Clés |
   392→| `activites-cles` | `activitesCles` | Activités Clés |
   393→| `ressources-cles` | `ressourcesCles` | Ressources Clés |
   394→| `proposition-valeur` | `propositionValeur` | Proposition de Valeur |
   395→| `relations-clients` | `relationsClients` | Relations Clients |
   396→| `canaux` | `canaux` | Canaux |
   397→| `segments-clients` | `segmentsClients` | Segments Clients |
   398→| `structure-couts` | `structureCouts` | Structure des Coûts |
   399→| `sources-revenus` | `sourcesRevenus` | Sources de Revenus |
   400→
   401→#### États du BMC
   402→
   403→```typescript
   404→type BmcStatus = 'DRAFT' | 'GENERATED' | 'REFINED'
   405→```
   406→
   407→| État | Signification |
   408→|------|---------------|
   409→| `DRAFT` | Aucune génération IA effectuée |
   410→| `GENERATED` | Canvas généré une fois depuis le BP |
   411→| `REFINED` | Blocs individuellement raffinés par l'IA |
   412→
   413→#### Actions IA
   414→
   415→1. **"Générer avec l'IA"** (depuis le BP) : `POST /api/bmc` avec `action: 'generate-from-bp'`  
   416→   → Récupère le contexte du BP + tous les simulateurs + parcours → génère les 9 blocs en un appel LLM → sauvegarde automatiquement
   417→
   418→2. **Suggestion par bloc** : `POST /api/bmc` avec `action: 'ai-suggest-block'` + `blockId`  
   419→   → Suggestion IA pour un seul bloc, l'utilisateur peut accepter ou modifier
   420→
   421→#### Export
   422→
   423→- **PDF** : `GET /api/export/bmc` → Page HTML A4 paysage (print-to-PDF) avec branding CreaPulse
   424→- **Export démo** : `GET /api/export/demo/bmc` → Même format sans authentification

   428→### 4.6 Business Plan
   429→
   430→**Fichier** : `src/components/bureau/modules/business-plan.tsx` (~1 000+ lignes)  
   431→**API** : `GET/PUT/POST /api/business-plan`  
   432→**LocalStorage** : `creapulse-bp`
   433→
   434→> **Le Business Plan est le HUB CENTRAL du pipeline Stratégie.** Toutes les données des autres modules convergent ici.
   435→
   436→#### Architecture de l'éditeur
   437→
   438→```
   439→┌─────────────────────────────────────────────────────────┐
   440→│  Sidebar gauche (sections)          │  Zone d'édition  │
   441→│  ┌─────────────────────────┐        │  principale      │
   442→│  │ ▼ Présentation (5)      │        │                   │

   849→## 7. Synchronisation inter-modules
   850→
   851→### Mécanisme de synchronisation
   852→
   853→La synchronisation entre modules fonctionne via le pattern **"Producteur → Consommateur via API"** :
   854→
   855→1. **Chaque simulateur** (Marché, Juridique, Financier, CreaSim) sauvegarde ses données dans sa propre table Prisma
   856→2. **Après chaque sauvegarde**, le simulateur appelle `POST /api/business-plan` avec `action: 'sync-simulators'`
   857→3. **L'API BP** récupère les données des 5 tables en parallèle (`Promise.all`) et les mappe aux sections BP
   858→4. **Fusion non-destructive** : les sections déjà remplies manuellement ne sont pas écrasées
   859→
   860→### Flux de synchronisation
   861→
   862→```
   863→┌──────────┐  save()   ┌──────────────────┐  read()   ┌──────────────┐
   864→│  Marché   │────────→│  MarketAnalysis    │────────→│  BP Section  │
   865→│  Module   │         │  (Prisma table)   │         │  etude-marche│
   866→└──────────┘         └──────────────────┘         └──────────────┘
   867→                                                           ▲
   868→┌──────────┐  save()   ┌──────────────────┐  sync()   ┌────┴─────────┐
   869→│Juridique │────────→│ JuridiqueAnalysis  │────────→│  Business    │
   870→│  Module   │         │  (Prisma table)   │         │  Plan API    │
   871→└──────────┘         └──────────────────┘         └──────────────┘
   872→                                                           ▲
   873→┌──────────┐  save()   ┌──────────────────┐  sync()          │
   874→│Financier │────────→│ FinancialForecast  │────────→         │
   875→│  Module   │         │  (Prisma table)   │                  │
   876→└──────────┘         └──────────────────┘                  │
   877→                                                           │
   878→┌──────────┐  save()   ┌──────────────────┐  sync()          │
   879→│ CreaSim  │────────→│ CreaSimSimulation  │────────→         │
   880→│  Module   │         │  (Prisma table)   │                  │

   906→## 8. Intégration IA
   907→
   908→### Pattern d'intégration
   909→
   910→Tous les modules utilisent le même pattern d'appel IA via `callZAI()` (sdk `z-ai-web-dev-sdk`) :
   911→
   912→```typescript
   913→// Pattern standard
   914→const result = await callZAI({
   915→  prompt: `...prompt avec contexte...`,
   916→  temperature: 0.7,
   917→  max_tokens: 2000,
   918→});
   919→
   920→if (result.success) {
   921→  const content = result.content;
   922→  // Utiliser parseJSONFromAI() pour les réponses JSON
   923→} else {
   924→  // Retourne un message d'indisponibilité (jamais d'erreur 500)
   925→  return aiUnavailableResponse();
   926→}
   927→```
   928→
   929→### Paramètres IA par module
   930→
   931→| Module | Température | Max Tokens | Format réponse |
   932→|--------|-------------|------------|---------------|
   933→| Business Plan (section) | 0.7 | 1 200 | Markdown |
   934→| Business Plan (parcours) | 0.7 | 3 000 | JSON |
   935→| BMC (complet) | 0.7 | 2 500 | JSON (9 blocs) |
   936→| BMC (par bloc) | 0.7 | 800 | Texte |
   937→| Pitch Deck (complet) | 0.7 | 2 500 | JSON (8 slides) |
   938→| Pitch Deck (par slide) | 0.7 | 800 | Texte |
   939→| Marché (analyse) | 0.7 | 800-2 000 | JSON / Texte |
   940→| Juridique (suggest) | 0.7 | 400 | JSON |
   941→| Juridique (autofill) | 0.7 | 600 | JSON |
   942→| Financier (synthèse) | 0.7 | 3 000 | Markdown (5 sections) |
   943→| Bilan IA | 0.5 | 2 000 | JSON |
   944→
   945→### Garde-fous IA
   946→
   947→- **Aucune erreur 500** : En cas d'indisponibilité de l'IA, un message graceful est retourné
   948→- **Non-destructif** : Les suggestions IA ne remplacent jamais le contenu existant (sauf demande explicite)
   949→- **Parse JSON** : Utilisation de `parseJSONFromAI()` pour extraire le JSON des réponses LLM
   950→- **Context enrichi** : Chaque prompt inclut le contexte du projet (titre, secteur, cible) pour des réponses pertinentes

   954→## 9. Calcul de progression
   955→
   956→### API `/api/progress`
   957→
   958→L'API retourne la progression sous cette forme :
   959→
   960→```json
   961→{
   962→  "parcours": {
   963→    "progress": 60,
   964→    "modules": {
   965→      "profil-createur": true,
   966→      "mon-projet": true,
   967→      "vision": true,
   968→      "riasec": true,
   969→      "kiviat": true,
   970→      "bilan-ia": false
   971→    }
   972→  },
   973→  "strategie": {
   974→    "progress": 45,
   975→    "modules": {
   976→      "marche": true,
   977→      "juridique": true,
   978→      "financier": false,
   979→      "creasim": false,
   980→      "bmc": false,
   981→      "business-plan": false,
   982→      "pitch-deck": false
   983→    }
   984→  },
   985→  "global": 51
   986→}
   987→```
   988→
   989→### Critères de complétion — Stratégie (7 modules)
   990→
   991→| Module | Critère de complétion |
   992→|--------|----------------------|
   993→| `marche` | `sector` ET `targetAudience` non vides dans `MarketAnalysis` |
   994→| `juridique` | `recommendedStatus` non vide dans `JuridiqueAnalysis` |
   995→| `financier` | `year1Revenue` ET `year1Expenses` non null dans `FinancialForecast` |
   996→| `creasim` | `monthlyRevenue` > 0 dans `CreaSimSimulation` |
   997→| `bmc` | ≥ 5 des 9 blocs non vides dans `BusinessModelCanvas` |
   998→| `business-plan` | `bpStatus` ∈ {DRAFT, REVIEW, VALIDATED, EXPORTED} ET `bpScore` > 50 |
   999→| `pitch-deck` | `ZeroDraft.content` contient ≥ 8 séparateurs `---` |
  1000→
  1001→### Formule de progression globale
  1002→
  1003→```
  1004→progression_globale = Math.round(progression_parcours × 0.4 + progression_stratégie × 0.6)
  1005→```
  1006→
  1007→**La stratégie compte pour 60%** de la progression globale, reflétant son importance dans le parcours créateur.
  1008→
  1009→### Indicateurs visuels
  1010→
  1011→- **Sidebar** : Cercle de progression par section (Parcours, Stratégie, etc.)
  1012→- **SectionOverview** : Cartes avec badges de complétion par module
  1013→- **Business Plan** : Barre de progression + score /100 + pipeline status badges
